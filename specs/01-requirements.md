# 01. 要件定義書

## 1.1 概要

自分用のタイムカードアプリ。複数のプロジェクトを横断して稼働時間を記録し、月末にプロジェクトごとの請求額（時給 × 稼働時間）を確認できる。複数端末（PC / スマホ）からブラウザでアクセスして利用する。

## 1.2 用語定義

| 用語 | 定義 |
|------|------|
| プロジェクト (Project) | 稼働時間を記録する単位。クライアント案件など。 |
| 打刻 (Punch / Time Entry) | 1 回の作業セッション。開始時刻と終了時刻を持つ。 |
| 既定時給 (Default Hourly Rate) | プロジェクトに紐づく標準の時給（JPY/h）。 |
| 月次レート (Monthly Rate) | 特定の年月における時給オーバーライド。 |
| 請求額 (Billing Amount) | `稼働時間（時間） × 適用時給` で算出する金額（JPY）。 |

## 1.3 ユーザー像

- 利用者は自分自身のみ（シングルユーザー前提だが、認証付き）。
- フリーランス / 副業的な使い方を想定。
- 月末にクライアント別に「今月いくら稼いだか」をすぐ確認したい。

## 1.4 ユーザーストーリー

### EPIC A: アカウント
- A-1: 利用者として、Google アカウントでログインできる。
- A-2: 利用者として、ログアウトできる。
- A-3: 自分のメールアドレス以外でのログインを拒否したい（許可リスト方式）。

### EPIC B: プロジェクト管理
- B-1: 利用者として、プロジェクトを作成・編集・アーカイブできる。
- B-2: 利用者として、プロジェクトに「名前」「カラー」「既定時給」「メモ」を設定できる。
- B-3: アーカイブしたプロジェクトは打刻対象から外れるが、過去データは閲覧できる。

### EPIC C: 打刻（タイムカード）
- C-1: 利用者として、プロジェクトを選んで「開始」ボタンで打刻を開始できる。
- C-2: 進行中の打刻が 1 つあるとき、「終了」ボタンで打刻を確定できる。
- C-3: 進行中打刻があるプロジェクトは画面上で明確に分かる（タイマー表示）。
- C-4: 利用者として、過去日時を指定して打刻を手動追加できる。
- C-5: 利用者として、既存打刻の開始/終了/メモ/プロジェクトを編集できる。
- C-6: 利用者として、打刻を削除できる（ソフトデリート）。
- C-7: 各打刻に作業メモ（自由記述、長文可）を紐付けられる。

### EPIC D: 月次集計・請求額
- D-1: 利用者として、月（YYYY-MM）を選んでプロジェクトごとの稼働時間と請求額を確認できる。
- D-2: 利用者として、特定プロジェクトの月次レートを設定・編集できる（未設定なら既定時給を適用）。
- D-3: 集計画面では「月内のすべてのプロジェクト合計」も表示する。
- D-4: 集計画面から、その月の打刻一覧（プロジェクト別）にドリルダウンできる。

### EPIC E: 横断機能
- E-1: スマホブラウザでも快適に打刻・閲覧できる。
- E-2: 同一アカウントで別端末から開いてもデータが同期される（クラウド DB）。

## 1.5 機能要件 (FR)

| ID | 要件 | 優先度 |
|----|------|--------|
| FR-01 | Google OAuth によるログイン / ログアウト | Must |
| FR-02 | 許可メールアドレスでのみログイン可能 | Must |
| FR-03 | プロジェクト CRUD（作成・一覧・編集・アーカイブ） | Must |
| FR-04 | プロジェクトの既定時給設定（JPY/h、整数） | Must |
| FR-05 | 月次レート（プロジェクト × 年月）の設定 | Must |
| FR-06 | リアルタイム打刻（開始 / 終了） | Must |
| FR-07 | 進行中打刻の同時 1 件制約（重複不可） | Must |
| FR-08 | 過去打刻の手動追加・編集・削除 | Must |
| FR-09 | 打刻ごとのメモ保存 | Must |
| FR-10 | 月次サマリー画面（プロジェクト別の稼働時間・請求額） | Must |
| FR-11 | 打刻一覧（フィルター：年月・プロジェクト） | Must |
| FR-12 | レスポンシブ UI | Must |
| FR-13 | CSV エクスポート | Should（将来拡張） |
| FR-14 | PWA 化 | Could（将来拡張） |

## 1.6 非機能要件 (NFR)

| ID | カテゴリ | 要件 |
|----|----------|------|
| NFR-01 | 性能 | 主要画面の TTI が 3G 4G 想定で 3 秒以内 |
| NFR-02 | 可用性 | Vercel / Supabase の SLA（サービスレベル合意＝稼働率などサービス品質を契約で定める指標）に準拠（自前で SLA は持たない） |
| NFR-03 | セキュリティ | Supabase RLS でユーザー単位データ分離 |
| NFR-04 | セキュリティ | OAuth トークンはサーバー側 Cookie（httpOnly）で管理 |
| NFR-05 | 監査 | 打刻の作成・更新・削除日時を保持 |
| NFR-06 | 国際化 | 通貨 JPY 固定 / タイムゾーン JST 固定（実装上は UTC で保存し JST で表示） |
| NFR-07 | コスト | 個人利用想定の範囲で完全無料運用 |
| NFR-08 | 保守性 | TypeScript の strict モード、ESLint / Prettier、ユニットテスト（最低限の集計ロジック） |
| NFR-09 | 移植性 | DB 接続はインターフェース化し、将来的な差し替えが可能 |

## 1.7 スコープ外（v1 では実装しない）

- マルチユーザー / チーム共有（将来検討）
- 請求書 PDF 出力 / 印刷
- 銀行 API / 入金連携
- 通知（リマインド・打刻し忘れアラート）
- Slack / カレンダー連携
- 多通貨 / 多タイムゾーン
- ネイティブアプリ

## 1.8 受け入れ基準（v1 リリース条件）

- [ ] Google ログインで自分のアカウントだけが入れる
- [ ] プロジェクトを 3 件以上作成し、それぞれで打刻ができる
- [ ] スマホ・PC 両方からアクセスして同じデータが見える
- [ ] 月次サマリー画面で「プロジェクト別の稼働時間・請求額・合計」が正しく表示される
- [ ] 月次レートを設定するとサマリーに即時反映される
- [ ] 打刻の手動追加・編集・削除ができる
- [ ] Lighthouse の Performance / Best Practices / Accessibility が 80 点以上
