# 07. 請求先・請求書（PDF）詳細仕様

> ヒアリング結果（2026-05）を反映。実装前に本書と `01-requirements.md` / `03-data-model.md` の整合を確認する。

## 7.1 スコープ

| 項目 | 方針 |
|------|------|
| 成果物 | **PDF ダウンロードのみ**（CSV は対象外） |
| 請求書番号 | **なし**（印字・DB ともに持たない） |
| 履歴 | **保存しない**。同じ月・同じ請求先でも **再生成・再ダウンロード可**（スナップショット無し） |
| インボイス登録番号 | **v1 では印字しない**（将来拡張の余地あり） |
| 対象期間 | **JST の暦月**（`YYYY-MM` の 1 日 0:00〜翌月 1 日 0:00 未満）。**締日カスタムは v1 では不要** |

## 7.2 ビジネスルール

### 7.2.1 プロジェクトと請求先

- 各 **プロジェクト** は **高々 1 つの請求先** に紐づく（`projects.billing_client_id`）。
- **複数プロジェクトを同一請求先にまとめる**ことは可。
- **同一プロジェクトを複数請求先に割り当て不可**（DB で表現上も 1 列参照のみ）。

### 7.2.2 対象プロジェクトなし

- 請求先に紐づくプロジェクトが **0 件**のとき、PDF 生成自体は **拒否せず実行可**。
- UI では **警告**（トーストまたはインライン）を出す。PDF は明細 0 行・合計 0 円でよい。

### 7.2.3 課税標準（税の前の金額）

- 各プロジェクト行の課税標準額（報酬ベース）は、既存の月次集計と同じく  
  `line_base = round_half_up(hours × applicable_rate)`（円、整数）。  
  ※ `applicable_rate` は当該 `year_month` の月次レートがあればそれ、なければ既定時給。
- 請求書集計でも **この金額を唯一の入力**とし、別途「税抜単価」からの再計算はしない。

## 7.3 消費税モード（請求先ごと）

消費税率は **10% 固定**。軽減税率・非課税行は **v1 では扱わない**。

請求先マスタに `tax_mode` を持つ。

### 7.3.1 外税（`exclusive`）

1. 各明細の「小計」= 当該行の `line_base`（**税抜**）。
2. `S = Σ line_base`（税抜合計）。
3. `tax = round_half_up(S × 0.1)`（**消費税額は請求書単位で 1 回**、円の四捨五入）。
4. `G = S + tax`（お支払額・税込合計）。

PDF には **税抜小計・消費税額・税込合計** を表示する。

### 7.3.2 内税（`inclusive`）

1. `S = Σ line_base`（課税標準＝税抜相当の合計。入力は既存ロジック由来）。
2. 請求の **税込合計** `T = round_half_up(S × 1.1)`（請求書単位、1 円四捨五入）。
3. 明細の「小計」欄は **税込** で表示し、**`Σ 明細小計 = T`** となるよう **最大剰余法（Largest Remainder Method）** で `line_base` の比率に応じて配分する（作業時間は変更しない）。
4. フッター表示:
   - **合計（税込）** = `T`
   - **（うち消費税等 ¥X）** として `X = T - S` を表示する。  
     ※ `X` は請求単位の四捨五入に伴い、厳密に `round(S × 0.1)` と一致しない場合がある。v1 は **表示上の内訳** として許容する。

一般的なインボイス形式の厳密な「税額」「本体」の組み合わせは将来拡張とし、v1 は上記で統一する。

## 7.4 請求先マスタに保持する項目（請求書に使うもの）

| 項目 | 必須 | PDF での扱い |
|------|------|----------------|
| 請求先名（会社名など） | Must | 宛先見出し |
| 件名（タイトル文） | Must | タイトル行。プレースホルダ可（下記） |
| 請求元の名前 | Must | 発行元の表示名 |
| 税区分（内税 / 外税） | Must | 7.3 の計算に使用 |
| PDF ファイル名テンプレート | Must（デフォルトあり） | ダウンロード時のファイル名生成 |
| 振込先 | Optional（v1） | 値がある場合のみ PDF にセクション表示 |
| メモ・備考（将来） | Optional | v1 では **フィールドのみ用意してもよい**（未印字可） |

### 7.4.1 件名・テンプレのプレースホルダ（推奨）

件名および必要なら他テキストで置換可:

| トークン | 例 | 意味 |
|----------|-----|------|
| `{YYYYMM}` | `202605` | 対象月 |
| `{YYYY-MM}` | `2026-05` | 対象月（ハイフン付き） |
| `{YYYY}` / `{MM}` | | 年・月（2 桁ゼロ埋め） |

未認識の `{...}` はそのまま残すか、実装で空文字にするかは実装時に統一（仕様上は **認識トークンのみ置換**）。

### 7.4.2 PDF ファイル名テンプレート

- 請求先ごとに **1 文字列**を保存する。
- 推奨デフォルト: `{YYYYMM}_{CLIENT}.pdf`
- 利用可能トークン（件名と同様）に加え、**`{CLIENT}`** = 請求先名からファイル安全な文字列へ正規化したもの（スラッシュ等除去、長さ上限あり）。

**例**: `{YYYYMM}_株式会社サンプル.pdf` → 実際は `202605_株式会社サンプル.pdf`（`CLIENT` が正規化済みの場合）。

拡張子 `.pdf` が無ければ実装側で付与してよい。

## 7.5 PDF の体裁（v1）

### 7.5.1 ヘッダ・メタ情報

- **請求日**: 対象月の **JST 暦月末日**（例: `2026-05` → `2026-05-31`）。
- **件名**: マスタの件名（プレースホルダ置換後）。
- **請求元**: マスタの「請求元の名前」。
- **請求先**: マスタの請求先名。
- **ロゴ（任意）**: `public/` に `invoice-logo.png` / `.jpg` / `.jpeg` / `.webp` のいずれかで置くと PDF ヘッダ左に **枠なし** で表示（無ければ非表示）。正方形に近い透過 PNG を推奨。

### 7.5.2 明細テーブル

列:

| プロジェクト名 | 作業時間 | 小計 |
|----------------|----------|------|

- **単価列は印字しない**（要件より）。
- 「小計」は税区分に応じ **税抜（外税）または税込（内税）**（7.3 参照）。
- 作業時間は **既存サマリーと同一の時間計算**（JST 月境界・オーバーラップ按分）。
- **対象月の稼働時間が 0 時間のプロジェクトは明細に含めない**（PDF・画面プレビュー共通）。消費税・合計は **明細に載る行のみ** を合算して算出する。

### 7.5.3 合計・税

- 外税: 税抜合計、消費税（10%）、税込合計。
- 内税: 税込合計、うち消費税等（7.3.2）。

### 7.5.4 振込先

- マスタに値がある場合のみ、本文末に「お振込先」等のブロックで表示。

## 7.6 例外・エラー

- 存在しない請求先 ID での生成: `NOT_FOUND`。
- 未ログイン: `UNAUTHORIZED`。
- PDF 生成ライブラリ失敗: `INTERNAL_ERROR`（ユーザーには汎用メッセージ）。

## 7.7 テスト観点（追加）

- 外税: `S` と `tax` の四捨五入が **請求 1 回**のみであること。
- 内税: 明細小計の和が **常に `T`** と一致すること（按分誤差なし）。
- プロジェクト 0 件: 警告が出ること、PDF が生成できること。
- 紐づくプロジェクトはあるが **すべて稼働 0 時間**の月: 明細は空、警告表示、PDF は明細 0 行で生成可。
