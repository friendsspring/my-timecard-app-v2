**仕様書駆動開発 (Spec-driven Development)** の方針に従い、まず仕様を確定させてから実装に着手します。

```
[1] 要件定義  →  [2] 設計  →  [3] レビュー（あなた）  →  [4] 実装  →  [5] テスト・デプロイ
       ↑                                  │
       └──────────── 必要に応じて修正 ─────┘
```

## 確定している前提条件（ヒアリング結果）

- **用途**: 自分用のタイムカード（プロジェクトを複数管理）
- **認証**: Google ログイン（OAuth）
- **打刻**: リアルタイム打刻 + 手動入力の両方対応
- **時給**: プロジェクトごとに既定時給を設定し、月単位でオーバーライド可能
- **請求額**: 画面上で集計表示（CSV/PDFは将来拡張）
- **対応端末**: レスポンシブ Web（PC / スマホブラウザ）
- **タイムゾーン**: JST 固定 / 通貨は JPY のみ
- **追加要件**: 各打刻にメモを紐付け
- **インフラ方針**: 無料枠最大化（Vercel + Supabase など）
- **言語**: フロント / バックエンド共に TypeScript
